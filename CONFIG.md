## RAG Integration Configuration

### ⚠️ CRITICAL: No External Dependencies Required

The RAG integration uses **ONLY Node.js built-in modules**:
- `http` - for HTTP requests (replaces axios)
- `fs` - for file operations
- `path` - for path handling
- Regex - for XML parsing (replaces xml2js)

**No npm packages needed**: Works in restricted corporate networks!

### Status Field Configuration

**⚠️ IMPORTANT - Case Sensitivity**:
- Backend sends: `'passed'` or `'failed'` (lowercase)
- RAG service expects: Case-insensitive comparison
- Statistics use: `.toLowerCase()` for comparison
- Filters use: `.toLowerCase()` for comparison

Valid status values (any case):
- `passed`, `Passed`, `PASSED`
- `failed`, `Failed`, `FAILED`

### TestType Field Configuration

**⚠️ IMPORTANT - Case Sensitivity**:
- Backend can send: `'selenium'`, `'functional'`, `'performance'` (any case)
- RAG service expects: Case-insensitive comparison
- Valid test types (any case):
  - `selenium`, `Selenium`, `SELENIUM`
  - `functional`, `Functional`, `FUNCTIONAL`
  - `performance`, `Performance`, `PERFORMANCE`

### Integration Flow

1. **Test Execution**:
   ```
   User Request → Frontend → Backend API
   → testExecutor.runTest()
   → Maven/TestNG execution
   → Test completion
   ```

2. **RAG Reporting (Automatic)**:
   ```
   Test completion → testExecutor (mvnProcess.on('close'))
   → ragIntegrationService.reportTestExecution()
   → Parse TestNG XML results
   → Extract steps and screenshots
   → HTTP POST to RAG server
   → RAG saves to database
   ```

3. **Data Collection**:
   - Test class and method names
   - Exit code and status
   - Duration in milliseconds
   - TestNG results (from `target/surefire-reports/testng-results.xml`)
   - Test steps (from console output `[STEP N]` markers)
   - Screenshots (from `[SCREENSHOT] Saved:` markers)

## API Endpoints

### Backend Server (8080)

#### Test Management
- `POST /api/generate-ai-test` - Generate and execute AI test
  - Body: `{ prompt, testType, llm }`
  - Returns: Test artifacts and execution ID

#### Test History
- `GET /api/history` - Get test execution history
- `POST /api/testcases/matches` - Match test cases

#### RAG Integration (Proxy to RAG Server)
- `GET /api/rag/stats` - Get test statistics from RAG
- `POST /api/rag/query` - Query test history with natural language

### RAG Server (8081)

#### Test History
- `GET /api/test-history` - List all test runs
  - Query params: `limit`, `testType`, `status`, `startDate`, `endDate`
- `GET /api/test-history/stats` - Get statistics
  - Returns: total, passed, failed, functional, performance, avgDuration, passRate
- `GET /api/test-history/:runId` - Get specific test run
- `POST /api/test-history` - Save new test run (auto-called by backend)

#### Natural Language Queries
- `POST /api/test-history/query` - Query history
  - Body: `{ question }`
